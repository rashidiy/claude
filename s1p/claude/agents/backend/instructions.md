## Import Convention

All code under `source/` (added to sys.path). Use bare module paths:


## Non-Negotiable Patterns

1. **Multi-tenant isolation** — Every query filters by `company_id`. No exceptions.
2. **Permission decorators** — `@require_permissions()` on all protected endpoints.
3. **Pydantic schemas** — Explicit request/response schemas. No raw dicts.
4. **Async everything** — All DB, HTTP, I/O uses async/await.
5. **Soft delete** — `deleted_at` timestamps, filter `deleted_at.is_(None)` in queries.
6. **Test pattern** — Every endpoint: happy path + error path + tenant isolation test.
7. **Migrations** — Alembic for every model change. Review autogenerated output.
8. **Error handling** — Proper HTTP codes (404, 403, 409, 422). Never 500 for expected errors.
9. **Configuration** — Through `core/config.py`. No hardcoded values.
10. **Background tasks** — For long-running I/O (email, webhooks, Telegram).

## Rules

- Never change auth/JWT logic without approval
- Never create tables without migration
- Never skip company_id filter
- Never return raw exceptions to client
- Never store secrets in code
- Never log PII (mask phones, emails)
- Never break existing endpoints

---

## Principles

1. Data model is king — get it right, everything follows.
2. Test the contract, not the implementation.
3. Fail fast — validate at the boundary, reject bad data immediately.
4. Idempotency — handle duplicate POSTs gracefully.
5. Every query has a company_id.
6. Background tasks for I/O — never make users wait.
7. Migrations are forward-only.
8. Test suite is the safety net.
